# 品类接口文档

本文档汇总 **TL 比价模块** 中与品类（`dict_categories`）增删改查及硬删除相关的 HTTP 接口。

> 通用约定见 [`后端接口文档.md`](./后端接口文档.md)。更细的比价等业务说明见 [`api.md`](./api.md)。

---

## 1. 接口总览

| 操作 | 方法 | 路径 | 摘要 |
|------|------|------|------|
| **查**（简要列表） | GET | `/tl/get_categories` | 按分组聚合 `品类id` + `品类名` |
| **查**（映射详情） | GET | `/tl/get_category_mapping` | 完整映射，含 `行id` |
| **增**（快速新建） | POST | `/tl/upload_variety` | 按品种名批量新建/恢复 |
| **增/改**（映射维护） | POST | `/tl/update_category_mapping` | 整组替换或追加别名 |
| **改**（单行） | POST | `/tl/update_category_row` | 按 `行id` 改名或设主名称 |
| **删**（整组软删） | DELETE | `/tl/delete_category` | `is_active=0` |
| **删**（单行软删） | DELETE | `/tl/delete_category_row` | `is_active=0` |
| **硬删**（整组） | DELETE | `/tl/purge_category` | 物理删除；须先软删；默认级联 |
| **硬删**（单行） | DELETE | `/tl/purge_category_row` | 物理删除；须先软删；默认级联 |

## 2. 硬删除接口（7e / 7f）

### 2.1 永久删除品类分组

| 项 | 值 |
|----|-----|
| 方法 | `DELETE` |
| 路径 | `/tl/purge_category` |
| Query | `品类id`（必填）、`cascade`（默认 `true`） |

**前置条件：** 该 `category_id` 下**全部别名须已软删除**（调用 `delete_category` 或逐条 `delete_category_row`）。仍有 `is_active=1` 的行时返回 **400**。

**`cascade=true`（默认）** 同一事务内按顺序清理：

1. `factory_demand_items`（组内 `row_id`）
2. `warehouse_inventories`（组内 `row_id`）
3. `warehouse_inventory_snapshots`（`category_id`）
4. `warehouse_category_receipt_prices`（`category_id`）
5. `quote_details`（组内全部 `name` 精确匹配）
6. `dict_categories`（该 `category_id` 全部行）

**`cascade=false`：** 仅当上述子表均无引用时才物理删字典行，否则 **409**。

**响应示例：**

```json
{
  "code": 200,
  "msg": "已永久删除品类分组 id=304，并清除关联数据（见 deleted_counts）",
  "cascade": true,
  "deleted_counts": {
    "factory_demand_items": 0,
    "warehouse_inventories": 0,
    "warehouse_inventory_snapshots": 2,
    "warehouse_category_receipt_prices": 1,
    "quote_details": 15,
    "dict_categories": 2
  }
}
```

### 2.2 永久删除单条品类别名

| 项 | 值 |
|----|-----|
| 方法 | `DELETE` |
| 路径 | `/tl/purge_category_row` |
| Query | `行id`（必填）、`cascade`（默认 `true`） |

**前置条件：** 该行 `is_active=0`（须先调用 `delete_category_row`），否则 **404**。

**`cascade=true`（默认）：**

1. 删该行关联的 `factory_demand_items`、`warehouse_inventories`
2. 删 `quote_details` 中 `category_name` 等于该别名 `name` 的记录
3. `DELETE dict_categories WHERE row_id=?`
4. 若删后该 `category_id` 下**无剩余行**，顺带删 `warehouse_inventory_snapshots` / `warehouse_category_receipt_prices`

**`cascade=false`：** 有子表引用则 **409**。

---

## 3. 推荐操作流程

```
软删除（可恢复）          硬删除（不可恢复）
─────────────────        ─────────────────
delete_category    →     purge_category
delete_category_row →    purge_category_row
```

硬删除会**永久清除**匹配的历史报价（`quote_details`），操作前请确认。

---

## 4. HTTP 状态码

| 状态码 | 场景 |
|--------|------|
| 200 | 删除成功 |
| 400 | 参数无效，或记录仍为启用状态（须先软删） |
| 404 | `品类id` / `行id` 不存在 |
| 409 | `cascade=false` 且仍有子表引用 |
| 500 | 服务端异常 |
